# คู่มือทดลองเล่นรอบแรก

เป้าหมายของรอบแรกคือดูว่าเด็กเข้าใจกติกา สนุก และผลัดกันเล่นบน iPad ได้หรือไม่ ไม่ใช่ทดสอบว่าเด็กเก่งคณิตศาสตร์เพียงใด

## กลุ่มทดลอง

- เด็ก ป.1 จำนวนอย่างน้อย 3 คน
- เด็ก ป.2 จำนวนอย่างน้อย 3 คน
- ทดลองทั้งคู่ 2 คนและกลุ่ม 4 คน
- มีครูสังเกต 1 คน โดยพยายามไม่อธิบายระหว่างเล่น

## ลำดับทดลอง

1. วาง iPad ให้เด็กและพูดเพียงว่า “ลองตั้งค่าแล้วแข่งกันให้จบหนึ่งรอบ”
2. เริ่มด้วยการบวกไม่เกิน 10 คนละ 3 ตา
3. ไม่ช่วยจนกว่าเด็กจะหยุดนิ่งหรือถามเอง
4. หลังจบรอบ ถามสั้น ๆ ว่า “ส่วนไหนสนุก”, “ส่วนไหนงง” และ “อยากเล่นอีกไหม”
5. หากเด็กพร้อม ให้ทดลองระดับไม่เกิน 20 อีกหนึ่งรอบ

## สิ่งที่ครูบันทึก

| หัวข้อ | ผ่านเมื่อ |
|---|---|
| ตั้งค่าเกม | เด็กเริ่มเกมได้โดยช่วยไม่เกิน 1 ครั้ง |
| เข้าใจตาเล่น | เด็กส่วนใหญ่รู้ว่าเป็นตาใครโดยไม่ต้องบอก |
| การกด | ไม่มีการกดผิดเพราะปุ่มเล็กหรืออยู่ไกลบ่อย ๆ |
| คำใบ้ | เด็กเข้าใจว่าคำใบ้ช่วยคิด ไม่ใช่เฉลยทันที |
| เวลารอ | ผู้เล่นที่รอไม่ออกจากเกมหรือแย่งกด |
| ความยุติธรรม | เด็กเข้าใจว่าทุกคนได้จำนวนตาเท่ากัน |
| ระยะเวลารอบ | รอบ 2 คนจบในประมาณ 3–5 นาที |
| เล่นซ้ำ | เด็กส่วนใหญ่เลือกเล่นอีกหนึ่งรอบ |

## คำถามสำคัญสำหรับครู

- เด็กเดาคำตอบเพื่อให้ตัวละครเดิน หรือพยายามคิดจริง?
- การให้ 2 ช่องเมื่อไม่ใช้คำใบ้ทำให้เด็กหลีกเลี่ยงคำใบ้หรือไม่?
- เด็กที่รอตาพยายามบอกคำตอบเพื่อนจนเกิดความขัดแย้งหรือไม่?
- ควรมีช่วงส่ง iPad ให้ผู้เล่นคนถัดไปชัดขึ้นหรือไม่?
- โหมด 4 คนใช้เวลารอนานเกินไปหรือไม่?

## เกณฑ์ก่อนเพิ่มฟีเจอร์

เดินหน้าสู่รอบถัดไปเมื่อปัญหาส่วนใหญ่เป็นเรื่องเนื้อหาเพิ่มเติม ไม่ใช่ความสับสนในการตั้งค่า การผลัดตา ปุ่มตอบ หรือกติกาคะแนน หากเด็กยังไม่เข้าใจ core loop ให้แก้ flow เดิมก่อนเพิ่มตัวละคร ด่าน หรือระบบรางวัล
