# Math Race Junior

## รูปแบบสนามแข่งขัน

- 4 คน: แบ่งหน้าจอเป็น 4 มุม แต่ละคนแตะคำตอบในพื้นที่สีของตัวเอง และใช้โจทย์/เวลาเดียวกันตรงกลาง
- 2 คน: แบ่งพื้นที่ตอบเป็นครึ่งบน–ล่าง เพื่อให้เด็กนั่งคนละฝั่งของ iPad ได้สะดวก
- ตอบผิด: พื้นที่ของผู้เล่นคนนั้นกระพริบสีแดงและพักตอบเฉพาะข้อนั้น
- เหลือไม่เกิน 10 วินาที: หน้าจอเริ่มเตือนสีแดง และกระพริบถี่ขึ้นเมื่อเหลือไม่เกิน 5 วินาที
- เสียงถูกปลดล็อกหลังการแตะปุ่มเริ่มเกมเพื่อรองรับข้อจำกัดเสียงของ Safari บน iPad

เกมคณิตศาสตร์แบบแข่งขันพร้อมกันบน iPad เครื่องเดียว สำหรับเด็ก ป.1–ป.2 รองรับผู้เล่น 1–4 คน โดยใช้โจทย์กลางหนึ่งข้อและแผงคำตอบแยกตามสีของผู้เล่น

## กติกา

- เลือกเวลาแข่งขัน 30 หรือ 60 วินาที
- ผู้เล่นทุกคนเห็นโจทย์เดียวกันและตอบพร้อมกัน
- คนแรกที่ตอบถูกได้ 1 คะแนน แล้วเกมขึ้นโจทย์ใหม่ต่อเนื่อง
- คนที่ตอบผิดจะถูกล็อกเฉพาะโจทย์นั้น แต่ผู้เล่นอื่นยังตอบต่อได้
- หากทุกคนตอบผิด เกมจะแสดงคำตอบแล้วไปข้อถัดไป
- ตัวเลือกของแต่ละแผงเรียงคนละตำแหน่ง ลดการกดตามกัน
- มีนับถอยหลัง 3–2–1 ก่อนจับเวลา
- สุ่มจากคลังโจทย์ทั้งหมดโดยไม่ซ้ำจนหมดชุด แล้วจึงสับชุดใหม่

โหมดนี้วัดทั้งความแม่นและความคล่อง จึงควรจับคู่เด็กที่ใช้ระดับโจทย์ใกล้เคียงกัน

## ฟีเจอร์ใน MVP

- เล่นคนเดียวหรือแข่งขันพร้อมกัน 2–4 คนบนเครื่องเดียว
- เลือกการบวก การลบ หรือแบบผสม
- ระดับไม่เกิน 10, ไม่เกิน 20, สองหลักไม่ทด/ไม่ยืม และสองหลักมีทด/มีการยืม
- แผงคำตอบ responsive สำหรับ iPad แนวตั้งและแนวนอน
- ตารางคะแนนสดและสรุปคะแนน ความแม่นยำ และเวลาตอบเร็วที่สุด
- คลังโจทย์ตั้งแต่ 66 ถึง 4,365 รูปแบบตามหมวดและระดับที่เลือก
- บันทึกการตั้งค่าล่าสุดในเครื่อง
- PWA และ offline cache สำหรับติดตั้งบนอุปกรณ์ที่รองรับ
- ไม่มีบัญชีผู้ใช้ โฆษณา หรือติดตามข้อมูลเด็ก

## เริ่มใช้งาน

โปรเจกต์ไม่มี dependency ภายนอก ใช้ Node.js 20 ขึ้นไป:

```bash
npm start
```

จากนั้นเปิด `http://127.0.0.1:4173`

ถ้าจะเปิดจาก iPad ที่อยู่ Wi-Fi เดียวกัน บน PowerShell ให้รัน:

```powershell
$env:HOST="0.0.0.0"; npm start
```

แล้วเปิดผ่าน IP ของคอมพิวเตอร์ การติดตั้ง PWA/offline บนอุปกรณ์จริงควรตรวจอีกครั้งหลังนำขึ้น HTTPS เช่น GitHub Pages

## ตรวจคุณภาพ

```bash
npm test
npm run check
```

ชุดทดสอบตรวจว่า:

- ไม่มีคำตอบติดลบหรือเกินระดับที่เลือก
- มีตัวเลือก 4 ค่าและมีคำตอบถูกเพียงค่าเดียว
- กติกาโจทย์สองหลักแบบทด/ยืมถูกต้อง
- ผู้เล่นทุกคนได้ตัวเลือกชุดเดียวกัน แต่ตำแหน่งคำตอบต่างกัน
- การจัดอันดับและกรณีคะแนนเสมอถูกต้อง
- ไม่มีโจทย์ซ้ำก่อนใช้ครบทั้งคลัง และไม่ซ้ำตรงรอยต่อเมื่อสับชุดใหม่

## โครงสร้างโปรเจกต์

```text
├── assets/                  ไอคอนแอป
├── docs/                    คู่มือทดลองกับเด็กและครู
├── scripts/dev-server.mjs   เซิร์ฟเวอร์สำหรับพัฒนา
├── src/app.js               หน้าจอ ตัวจับเวลา และ game loop
├── src/question-engine.js   กฎสร้างโจทย์และตัวเลือก
├── src/question-pool.js     คลังโจทย์และการสุ่มแบบไม่ซ้ำ
├── src/race-rules.js        กติกาแผงคำตอบและการจัดอันดับ
├── tests/                   automated tests
├── index.html
├── styles.css
└── service-worker.js
```

ก่อนขยายฟีเจอร์ ให้ทดลองตาม [คู่มือ playtest](./docs/playtest-guide.md) และตัดสินใจจากพฤติกรรมเด็กจริง
